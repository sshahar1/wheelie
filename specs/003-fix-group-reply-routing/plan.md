# Implementation Plan: Fix Group Reply Routing

**Branch**: `003-fix-group-reply-routing` | **Date**: 2026-07-04 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/003-fix-group-reply-routing/spec.md`

## Summary

The bot currently always replies to `message.from` (the individual sender), so every
confirmation, clarifying question, query answer, and cancellation is DM'd privately instead of
posted into the troupe's shared WhatsApp group — violating FR-003 of the original feature.
Meta's Cloud API includes a `group_id` field on inbound group messages (`from` is always the
individual participant, never the group). The fix: capture `group_id` when present, and when
replying, send with `recipient_type: "group"` / `to: <group_id>` instead of `to: <from>`; fall
back to today's 1:1 behavior when no `group_id` is present. No persistence, schema, or
roster/activation/extraction logic changes are needed — this is a routing-destination fix
that flows entirely through the existing per-request message-handling pipeline.
